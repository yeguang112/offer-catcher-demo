# 5 个作业的产品梳理与落地路线

## 总体判断

如果目标是先做出一个最稳、最容易被评委快速理解的 Demo，优先顺序建议如下：

1. 作业一「Offer 捕手」
2. 作业二「未来鹅」
3. 作业四「实习能量站」
4. 作业五「自定义赛道」
5. 作业三「成长副本」

原因是：作业一最聚焦、用户价值最直观、产出也最容易形成闭环；作业二和四更适合做成长陪伴与多角色协同；作业五最能体现产品定义能力，但对问题选择要求高；作业三更偏内部学习路径工具，展示效果容易偏“管理后台化”。

---

## 作业一「Offer 捕手」

### 核心问题

- 学生不会筛岗位，海投低效
- 学生不会判断简历与目标岗位的差距
- 学生知道想投什么，但不知道怎么改简历更容易过筛

### MVP 方案

- 输入简历或基础信息
- 自动抽取候选人画像
- 对多个岗位做匹配排序
- 点击某个岗位后输出匹配理由与差距项
- 输出一份简历优化清单

### 差异化亮点

- 不是泛职业咨询，而是“岗位推荐 + 简历优化”双闭环
- 结果可解释，适合答辩展示
- 很容易扩展成真实 SaaS / Agent 产品

### 最适合的 Demo 形式

- Web 单页应用

## 作业二「未来鹅」

### 核心问题

- 低年级学生对企业、岗位、成长路径认知不足
- 企业与学生之间缺少持续陪伴式触达
- 校招信息触达单向，缺少个性化路径引导

### MVP 方案

- 按年级进入不同成长模式
- 大一看行业认知与公司地图
- 大二看专业与岗位映射
- 大三看实习建议与岗位准备
- 大四/研一看正式求职陪伴

### 差异化亮点

- 从“招聘”转成“成长陪伴”
- 有情绪价值和品牌价值
- 适合接入企业文化、员工故事、岗位介绍等内容库

### 最适合的 Demo 形式

- 陪伴式对话产品 + 可视化成长地图

## 作业三「成长副本」

### 核心问题

- AI Native 组织中的新人需要快速建立 AI 协作方式
- 传统培训只讲制度和流程，不讲人机协作
- HR 缺少可复用的 30-60-90 学习框架

### MVP 方案

- 输入岗位类别与新人基础
- 输出 30 / 60 / 90 天学习路径
- 每阶段给出目标、任务、AI 工具建议、成果物与评估标准
- 为导师与新人各生成一个版本

### 差异化亮点

- 强调 AI Native 组织语境
- 适合展示结构化能力
- 可以沉淀成组织培训资产

### 风险点

- 如果只做计划表，会显得像普通模板工具
- 需要加入 AI 协作方式设计，才有新意

### 最适合的 Demo 形式

- 生成式路径设计器 + 看板式输出

---

## 作业四「实习能量站」

### 核心问题

- 导师带教不标准
- 实习生不知道当前阶段该学什么
- HR 与业务方缺少统一进度视图

### MVP 方案

- 为实习生生成周度成长路线
- 为导师提供标准化带教提醒
- 为 HR 提供适岗进度看板
- 输出风险提醒，如掉队、无反馈、目标不清

### 差异化亮点

- 多角色协同非常清晰
- 很适合展示“AI + 看板”的实用价值
- 可以把成长、协同、评估放到一个工具里

### 最适合的 Demo 形式

- 智能看板 / Dashboard

---

## 作业五「自定义赛道」

### 核心问题

- 最重要的不是做什么，而是选一个足够具体、足够真实的问题

### 推荐切入方向

- 新员工入职 30 天反馈助手
- 内部转岗机会匹配助手
- 绩效反馈改写与情绪风险提示助手
- 员工敬业度访谈摘要与洞察助手

### 最推荐方向

内部转岗机会匹配助手。因为它和作业一类似，容易复用“画像 - 机会匹配 - 差距建议”的产品结构，同时又更 HR 场景化。

### 最适合的 Demo 形式

- Web 应用 / 智能体

---

## 从想法到落地的统一步骤

1. 先明确一个核心用户和一个核心任务，不要同时服务太多人。
2. 把真实场景拆成输入、判断、输出三层。
3. 定义 MVP，只保留最能证明价值的 1 个闭环。
4. 决定哪些地方用 AI，哪些地方用规则或静态数据。
5. 做一个可运行 Demo，而不是只做 PPT。
6. 补方案说明文档、部署链接和演示视频。

## GitHub 与公网部署统一步骤

1. 在项目目录执行 `git init`
2. 执行 `git add .`
3. 执行 `git commit -m "feat: initial demo"`
4. 到 GitHub 创建仓库
5. 执行 `git remote add origin your-repository-url`
6. 执行 `git branch -M main`
7. 执行 `git push -u origin main`
8. 到 Vercel / Netlify 导入 GitHub 仓库
9. 配置构建命令与输出目录
10. 拿到公网链接后放进最终提交材料
