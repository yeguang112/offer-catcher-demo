# 作业一方案说明

项目名称：Offer 捕手

问题诊断：
学生求职时通常有两个连续痛点。第一是岗位太多，不知道哪些值得优先投；第二是即使看中了岗位，也不知道简历差距在哪里，容易海投低效。核心不是信息缺失，而是缺少“个体画像”和“岗位要求”之间的翻译层。

方案设计：
产品设计成一个双阶段智能体。第一阶段做岗位匹配，把学生的专业、城市偏好、能力标签、项目经历、职业兴趣结构化后，与岗位 JD 做相似度与证据匹配，输出推荐清单、匹配理由和差距提示。第二阶段做简历优化，围绕目标岗位提取关键词、能力要求和经历证据，给出可执行的改写建议，例如补哪些关键词、重写哪些项目、如何强化量化结果和岗位叙事。

AI 工具选型理由：
大模型适合处理非结构化文本，如简历、JD、项目描述和求职意向，能完成信息抽取、语义对齐、改写建议生成；规则引擎适合处理稳定维度，如城市、届别、岗位类型、基础关键词覆盖，因此 Demo 采用“规则打分 + AI 解释”的思路，更稳定也更容易展示。

关键配置：
MVP 版本优先覆盖 3 个模块：候选人画像、岗位推荐、简历优化。前端采用 React + Vite，适合快速搭建公网可演示网页。后续若接入真实 AI，可增加简历上传解析、JD 粘贴分析、定制版简历一键生成和投递记录追踪。

迭代记录：
第一版只做聊天问答会显得泛，因此改为可视化决策面板。第二版加入岗位排序和分数解释，让推荐更可信。第三版加入“行动清单”，把分析结果转成可执行输出，增强交付感。

效果评估：
可用三个指标衡量：岗位浏览到投递的决策时间是否缩短；学生是否能更快确定主投方向；简历修改后关键词覆盖和叙事聚焦是否提升。对评委而言，这个方案的价值在于它不是泛泛的职业建议，而是直接帮助学生完成“选岗”和“改简历”两个关键动作。
